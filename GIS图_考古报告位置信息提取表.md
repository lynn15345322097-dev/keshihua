# GIS图：考古报告位置信息提取表

目标：先从《隋唐洛阳城 1959-2001年考古发掘报告》第二册中提取适合 GIS 落图的信息。此表暂不处理古籍解释、星象叙事和视觉设计，只处理“哪些对象能画成点、线、面”。

## 1. 提取原则

1. 优先提取带现代参照的对象，例如道路、学校、单位、河渠、厂区。
2. 优先提取带图号、比例尺、方向、探方编号的对象。
3. 把“实测/发掘/钻探/复原”分开，不把所有遗址画成同一种点。
4. 先做可配准的硬骨架，再补叙事节点。

## 2. GIS字段建议

| 字段 | 含义 |
|---|---|
| object_id | GIS对象编号 |
| object_name | 遗址/构件/探方名称 |
| geometry_type | 点、线、面、点组、图件 |
| ancient_layer | 古代空间层级 |
| period | 时期 |
| evidence_type | 发掘、钻探、图件复原、文献复原 |
| evidence_grade | A、B、C |
| modern_anchor | 报告中的现代参照 |
| ancient_position | 报告中的古代方位 |
| metric_info | 尺寸、方向、相对距离 |
| source | 页码、图号、探方号 |
| gis_method | GIS处理方式 |
| status | 当前处理状态 |

## 3. 证据等级

| 等级 | 标准 | GIS画法 |
|---|---|---|
| A | 有发掘/探方/明确现代参照，且图件可配准 | 实线、实心点、较高透明度面 |
| B | 有钻探或现代地点，但位置仍需相对推算 | 虚线、空心点、半透明面 |
| C | 主要来自复原图或文献位置关系 | 弱化符号、低透明度、不可作为硬边界 |

## 11. 当前判断

第一张 GIS 图不应从“七天建筑”开始，而应从“宫城范围 + 城墙 + 城门 + 探方”开始。原因是这批对象最容易从考古报告转成空间数据，也最适合作为后续古今叠加的基准层。

最优先配准的图件顺序：

1. 图5-1：宫城城墙发掘位置图，决定宫城大框架。
2. 图5-30：应天门1960年钻探遗迹分布图，决定中轴线南端关键锚点。
3. 图5-50：崇庆门GT245图，决定大内西南角附近位置。
4. 图5-53至图5-56：宣政门GT819图，决定大内东南角附近位置。
5. 图5-64：大内分区及城墙发掘位置图，决定大内内部框架。
6. 图5-65至图5-70：大内东墙、西墙各发掘点图，适合校正大内两侧墙线。
7. 图5-73：宫城大内街道发掘位置图，决定明德门内大街及其他路土点组。
8. 图5-91：宫城大内中区发掘探方分布图，同时含定鼎北路、中州路、定鼎南路、应天门、大内南墙、东西廊，是大内中区最重要的配准图。
9. 图5-90：宫城大内中区轴线建筑钻探遗迹分布图，决定隋唐/北宋轴线建筑基址组。
10. 图5-113：乾元门遗址发掘探方及唐代遗迹分布图，决定乾元门局部形态。
11. 图5-17：宫城东墙DT50-17图，适合校正东墙中段。
12. 图5-21、图5-22：宫城西墙GT442、GT795图，适合校正西墙南段和中南段。
